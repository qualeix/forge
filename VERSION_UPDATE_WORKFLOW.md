# Version Update Workflow

## Overview
This document outlines the standardized procedure for updating the Forge app version across all components of the project.

## Source of Truth
**`app.json`** is the single source of truth for version information:
- `"expo.version"`: Version string (e.g., "1.8.1")
- `"expo.android.versionCode"`: Version code (e.g., 25)

## Files That Reference Version

### 1. Configuration Files (Updated Automatically)
- **`app.json`** - Source of truth (manual update required)
- **`android/app/build.gradle`** - Auto-generated by `plugins/rename-apk.js`
  - `versionCode`: Read from `app.json.expo.android.versionCode`
  - `versionName`: Read from `app.json.expo.version`
  - `outputFileName`: Set to `forge-v${version}.apk`

### 2. Application Files (Manual Update Required)
- **`constants/updater.ts`** - `APP_VERSION` constant
- **`app/(tabs)/settings.tsx`** - Version display in UI

### 3. Plugin Configuration
- **`plugins/rename-apk.js`** - Reads from `app.json` and injects into build.gradle
- **`app.json`** - `expo-build-properties` plugin configuration

## Version Update Procedure

### Step 1: Update app.json
```json
{
  "expo": {
    "version": "X.Y.Z",           // Increment version string
    "android": {
      "versionCode": N             // Increment version code (usually +1)
    }
  }
}
```

### Step 2: Update Application Files
Update these files to match the new version:

**`constants/updater.ts`:**
```typescript
export const APP_VERSION = "X.Y.Z";
```

**`app/(tabs)/settings.tsx`:**
```tsx
<Text>X.Y.Z</Text>
```

### Step 3: Clean Build
⚠️ **CRITICAL WARNING**: `npx expo prebuild --clean` destroys all native modifications!

**Before running prebuild, backup these files:**
- `android/app/src/main/java/com/forge/app/BatteryModule.*` (Battery package)
- `android/app/src/main/AndroidManifest.xml` (manual permissions)
- Any other native Android modifications

**Safe Update Process:**
```bash
# Option 1: Incremental update (recommended for minor versions)
npx expo prebuild  # Without --clean to preserve native changes

# Option 2: Full clean (only for major versions or when native files are broken)
# 1. Backup native modifications first
# 2. Then run clean prebuild
npx expo prebuild --clean
# 3. Restore native modifications
```

**When to use --clean:**
- Major version upgrades
- Broken native builds
- When you want to reset to template
- **NOT** for routine version updates

- **Architecture**: arm64-v8a only

## Version Numbering Rules

### Version String (Semantic Versioning)
- Format: `MAJOR.MINOR.PATCH`
- **MAJOR**: Breaking changes
- **MINOR**: New features (backward compatible)
- **PATCH**: Bug fixes

### Version Code
- Increment by 1 for each version
- Must be higher than previous version
- Used by Google Play Store for update ordering

## Verification Checklist

After updating version:

### Pre-Build Checks
- [ ] `app.json` version and versionCode updated
- [ ] `constants/updater.ts` APP_VERSION updated
- [ ] `app/(tabs)/settings.tsx` version display updated
- [ ] If using `--clean`: Backup native modifications first
  - [ ] BatteryModule files backed up
  - [ ] AndroidManifest.xml backed up
  - [ ] Other native changes backed up

### Build Process
- [ ] Run appropriate prebuild command (`npx expo prebuild` or `npx expo prebuild --clean`)
- [ ] If clean build: Restore native modifications
- [ ] Verify plugin output shows correct version reading
- [ ] Build release APK successfully

### Post-Build Verification
- [ ] APK named correctly: `forge-vX.Y.Z.apk`
- [ ] APK size is ~39MB (arm64-v8a only)
- [ ] Test app displays correct version in settings
- [ ] BatteryModule functionality preserved (if applicable)
- [ ] Android permissions preserved (if manually added)

## Troubleshooting

### If APK name is incorrect:
1. Check `plugins/rename-apk.js` is reading version correctly
2. Run `npx expo prebuild --clean` to regenerate build.gradle
3. Verify build.gradle contains correct version

### If multiple architectures are built:
1. Verify `app.json` has correct `expo-build-properties` configuration
2. Check `plugins/rename-apk.js` has `splits` block
3. Run clean prebuild

### If native modifications are lost after prebuild:
1. **Always use incremental prebuild for version updates:**
   ```bash
   npx expo prebuild  # Without --clean
   ```
2. **If you used --clean accidentally:**
   - Restore from backup
   - Re-apply native changes manually
   - Test BatteryModule functionality
   - Verify Android permissions in manifest

### If versionCode doesn't update:
1. Check `app.json` android.versionCode value
2. Verify `plugins/rename-apk.js` is reading versionCode correctly
3. Regenerate build.gradle with clean prebuild

## Automation Notes

The following are automated by the plugin system:
- Reading version from `app.json`
- Injecting version into `build.gradle`
- Setting APK filename format
- Configuring architecture splits

The following require manual updates:
- Version constants in TypeScript files
- UI version display
- `app.json` version values (source of truth)
